# تطبيق ميزان القانوني — دليل التشغيل والتسليم

تطبيق استشارات قانونية سعودي مبني بـ **React Native (Expo)** + **Supabase Edge Function**.
استرشادي بالذكاء الاصطناعي، عربي/إنجليزي بالكامل عبر محرك لغوي مركزي.

---

## ١. تشغيل التطبيق (Expo)

المتطلبات: Node.js 18+ وحساب Expo (مجاني) وتطبيق **Expo Go** على هاتفك.

```bash
# 1) تثبيت الحزم (مرة واحدة)
npm install

# 2) تشغيل خادم التطوير
npx expo start
```

سيظهر رمز QR في الطرفية. افتح **Expo Go** على هاتفك وامسح الرمز ليعمل التطبيق مباشرة.
- أندرويد: امسح من داخل تطبيق Expo Go.
- آيفون: امسح بكاميرا النظام ثم افتح في Expo Go.

لبناء ملف تثبيت فعلي (APK للأندرويد) راجع `build-apk-guide.md`.

---

## ٢. ربط مفاتيح Supabase

التطبيق يتصل بـ Supabase في موضعين، وكلاهما يستخدم **المفتاح العام فقط** (publishable). المفاتيح السرية (OpenAI، Tavily، Service Role) تبقى على الخادم حصراً.

### أ) داخل التطبيق
الملف: `src/services/supabaseClient.js`
- `SUPABASE_URL`: رابط مشروعك في Supabase.
- `SUPABASE_ANON_KEY` (المفتاح العام publishable).

### ب) رابط الخادم (Edge Function)
الملف: `src/lib/api.js`
- `PROXY_URL`: رابط دالة `rapid-function` في مشروعك، بالشكل:
  `https://<معرّف-مشروعك>.supabase.co/functions/v1/rapid-function`

### ج) مفاتيح الخادم السرية
تُضبط في Supabase عبر: **Edge Functions ← Secrets**، وليست في كود التطبيق:
`OPENAI_KEY`, `TAVILY_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ADMIN_PASS`.
كود الخادم في `server/index.ts`. خطوات النشر في `server-setup-guide.md`.

### د) جداول قاعدة البيانات
نفّذ مرة واحدة: افتح **SQL Editor** في Supabase ثم الصق محتوى `database-setup.sql` ثم Run.
ينشئ: `app_settings` (الإعدادات والحصص)، `error_reports` (أعطال تقنية بلا هوية)،
و`user_consents` (سجل الموافقة القانونية — يُكتب تلقائياً عند التسجيل، محمي بـ RLS لكل مستخدم).

## ٣. المحرك اللغوي المركزي — `src/lib/i18n.js`

كل نصوص الواجهة في مكان واحد. لا نص مكتوب يدوياً داخل الشاشات؛ كل شيء عبر `t('key', lang)`.

### البنية
```js
const STRINGS = {
  ar: { key: "النص العربي", ... },   // النسخة العربية
  en: { key: "English text", ... },  // النسخة الإنجليزية
};
export function t(key, lang = "ar", vars) { ... }
```
- كل مفتاح موجود في `ar` و`en` بنفس الاسم.
- المفاتيح منظّمة بمجموعات معلّقة بالعربي (عام، تبويبات، بحث، بطاقات، محادثة، منصات، حاسبات، أقسام، مولّدات، معرفة، تذكيرات، قوالب، مقارنة، توقيع، مكتبة، لوحة، اشتراكات، إعدادات، مصادقة، الموافقة القانونية، الأشهر، التنبيهات).

### تعديل نص موجود
ابحث عن المفتاح في `i18n.js`، وعدّل قيمته في `ar` و/أو `en`. لا تحتاج لمس أي شاشة.

### إضافة نص جديد
1. أضف المفتاح في كائني `ar` و`en`:
   ```js
   // في ar
   my_new_key: "نص جديد",
   // في en
   my_new_key: "New text",
   ```
2. استخدمه في الشاشة: `{t("my_new_key", lang)}`.

### حقن متغيرات داخل النص
استخدم `{name}` في النص ومرّر القيمة:
```js
// في i18n: chat_welcome_section: "أهلاً بك في قسم {name}."
t("chat_welcome_section", lang, { name: "المرشد القانوني" })
```
هكذا تبقى اللغتان نظيفتين بلا خلط: القالب الإنجليزي يحقن قيمة إنجليزية، والعربي عربية.

### إضافة لغة جديدة (مثلاً الفرنسية)
1. أضف `fr: { ... }` داخل `STRINGS` بنفس المفاتيح.
2. أضف اللغة لمصفوفة `LANGUAGES`:
   ```js
   { code: "fr", label: "Français", dir: "ltr" },
   ```
3. ستظهر تلقائياً في مبدّل اللغة بالإعدادات.

> ملاحظة: تعليمات الذكاء الاصطناعي (الدساتير) ليست في `i18n.js` — تبقى في الخادم (`server/index.ts`) بالعربية، والمخرجات القانونية عربية حصراً مهما كانت لغة الواجهة.

## بنية المشروع المختصرة

```
App.js                      نقطة الدخول، التنقّل، البوابة
app.json / eas.json         إعداد Expo والبناء
server/index.ts             دالة الخادم (الذكاء الاصطناعي + البحث)
admin.html                  لوحة التحكم (الحصص والميزات)
src/
  lib/                      المنطق: i18n.js, theme.js, api.js, calculators.js, searchIndex.js ...
  services/supabaseClient.js  اتصال Supabase
  components/               DaawaWizard.js, ErrorBoundary.js
  screens/                  17 شاشة (الرئيسية، المحادثة، الحاسبات، المنصات ...)
database-setup.sql      جداول Supabase
```

الأدلة التفصيلية: `owner-guide.md`، `operation-catalog.md`، `store-publishing-guide.md`، `server-setup-guide.md`.

## بروتوكول الموافقة القانونية

عند التسجيل، تظهر نافذة إجبارية بالشروط (عربي/إنجليزي). زر «أوافق» معطّل حتى يمرّر المستخدم لنهاية النص. عند الموافقة يُسجَّل `terms_accepted` و`terms_accepted_at` في جدول `user_consents`، ولا يدخل المستخدم للتطبيق إلا بعد نجاح هذا التوثيق. هذا سجل امتثال يحمي قانونياً، وليس بيانات استخدام.
